## Jak s dokumentaci pracovat

- `Root Spec` je hlavni zdroj pravdy pro MVP.
- `Module Spec` rozepisuje jednu oblast do provoznich, UX a datovych detailu.
- `Implementation Spec` se ma vytvaret az z hotovych module specu.
- Prvni implementation spec pro MVP je `implementation-spec-auth-core-mvp.md`.
- Druhy implementation spec pro MVP je `implementation-spec-client-access-management-mvp.md`.
- Treti implementation spec pro MVP je `implementation-spec-client-profile-and-team-mvp.md`.
- Ctvrty implementation spec pro MVP je `implementation-spec-campaign-request-flow-mvp.md`.
- Paty implementation spec pro MVP je `implementation-spec-campaign-operations-and-review-mvp.md`.
- Pokud vznikne nova obchodni logika, nejdriv se ma propsat do `Root Spec` nebo `Module Spec`, ne rovnou do kodu.
