## Jak s dokumentaci pracovat

- `Root Spec` je hlavni zdroj pravdy pro MVP.
- `Module Spec` rozepisuje jednu oblast do provoznich, UX a datovych detailu.
- `Implementation Spec` se ma vytvaret az z hotovych module specu.
- Prvni implementation spec pro MVP je `implementation-spec-auth-core-mvp.md`.
- Druhy implementation spec pro MVP je `implementation-spec-client-access-management-mvp.md`.
- Treti implementation spec pro MVP je `implementation-spec-client-profile-and-team-mvp.md`.
- Ctvrty implementation spec pro MVP je `implementation-spec-campaign-request-flow-mvp.md`.
- Paty implementation spec pro MVP je `implementation-spec-campaign-operations-and-review-mvp.md`.
- Sesty implementation spec pro MVP je `implementation-spec-client-inbox-and-internal-notifications-mvp.md`.
- Sedmy implementation spec pro MVP je `implementation-spec-campaign-reporting-and-client-updates-mvp.md`.
- Osmy implementation spec pro MVP je `implementation-spec-campaign-extension-requests-mvp.md`.
- Devaty implementation spec pro MVP je `implementation-spec-admin-dashboard-and-operational-summary-mvp.md`.
- Desaty implementation spec pro MVP je `implementation-spec-campaign-email-notifications-mvp.md`.
- Jedenacty implementation spec pro MVP je `implementation-spec-client-dashboard-summary-mvp.md`.
- Dvanacty implementation spec pro MVP je `implementation-spec-email-delivery-history-and-admin-visibility-mvp.md`.
- Trinacty implementation spec pro MVP je `implementation-spec-client-notification-preferences-mvp.md`.
- Ctrnacty implementation spec pro MVP je `implementation-spec-global-email-oversight-mvp.md`.
- Patnacty implementation spec pro MVP je `implementation-spec-unsubscribe-links-and-off-app-preferences-mvp.md`.

- Sedmnacty implementation spec pro MVP je `implementation-spec-notification-preference-visibility-in-client-preview-mvp.md`.
- Pokud vznikne nova obchodni logika, nejdriv se ma propsat do `Root Spec` nebo `Module Spec`, ne rovnou do kodu.
